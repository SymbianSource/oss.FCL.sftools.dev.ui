Tasks view


The Tasks view allows you to view tasks that have been created as reminders actions you need to take while customizing the theme. You can also add Tasks to the list.


Figure: Task View
 
For more information adding and using tasks see Using Tasks

A task has the following properties:
  • Status — Defines whether the task is complete or active.
  • Description — A description of the task. If the task was created from a component the component name is used by default, you you can add your own text here if you wish.
  • Priority — The priority of the task recorded as High, Normal, or Low.
  • Resources — The theme file which the task applies to. This property is not use if you add a task from the Tasks view.
  • Path — Read only
  • Location — These properties define the location for the component for which you have marked the task. These properties are not in use if you add a task from the Tasks view. Read only

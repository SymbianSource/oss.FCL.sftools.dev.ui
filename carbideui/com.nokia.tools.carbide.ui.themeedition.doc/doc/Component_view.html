Component View

The Component view can be accessed from the Resources view. It displays all the elements that can be customized for a specific category. The Component view's name in the tab is the same as the category selected in the Resources view.

The figure below illustrates Carbide.ui with Context Pane Icons component view open.


Figure: Context Pane Icons

Note:  In the figure, the component view icons are displayed with title texts. To save space and to allow more icons to be displayed in a smaller area, click the Toggle title texts button. This will toggle off the title texts.


When you click on a category in the Resources view, the components view is refreshed with all the elements under that category including the elements under its sub-categories. Clicking on a sub-category displays elements specific to the sub-category only. Clicking on one of the elements in the Components View refreshes the Editor with the screen that has a preview for the selected element. Some elements, which do not have a preview, display an information dialog like in the image below and do not refresh the Editor. The pop-up is displayed for all elements that do not have a preview. It can be disabled by clicking on the checkbox which then displays this information only in the status bar at the bottom.


Figure: Information Dialog for Elements that do not have a preview


Selection of multiple components using CTRL key, CTRL-A or multiple select action

This feature helps select multiple elements from a theme by holding the CTRL key or using CTRL-A (to select all the elements at once).This is used in order to copy and paste several elements between various themes.
